# 1. この章でやること
この章では、**nix-darwin で Homebrew を管理し、ツールを入れる方法**を解説します。

:::message
すでに Homebrew がインストールされている前提の解説です。
nix-darwin は Homebrew 本体をインストールしません。
未インストールの場合は、[公式サイト](https://brew.sh/ja/)に従って導入してください。
:::

:::details Homebrew 本体も nix-darwin で管理したい場合
[nix-homebrew](https://github.com/zhaofengli/nix-homebrew) を nix-darwin のモジュールとして利用すると、Homebrew 本体のインストールも可能になります。
本書では扱いません。
:::


# 2. homebrew を有効化する
`configuration.nix` に homebrew の設定を追加します。

`<username>` は自身のユーザ名に置き換えてください。

```nix:~/work/dotfiles/configuration.nix
{
  homebrew = {
    enable = true;
    user = "<username>";
    brews = [
    ];
    casks = [
    ];
  };
}
```


# 3. ツールを追加する
`brews` は CLI ツール、`casks` は GUI アプリを指定します。
必要なツールを配列に追加していきます。

```nix:~/work/dotfiles/configuration.nix
  homebrew = {
    enable = true;
    user = "<username>";
    brews = [
      # brew install nginx
      "nginx"
    ];
    casks = [
      # brew install --cask microsoft-office
      "microsoft-office"
    ];
  };
```


# 4. その他の設定
## 4.1 homebrew.onActivation.cleanup
uninstall を指定すると、リストに記載していないツールは自動的にアンインストールされます。
既存の Homebrew で入れたツールを nix-darwin 側に全て移してから、設定してください。

移行が終わるまでは、未設定にしておくか、デフォルト値である none を指定してください。

## 4.1 ツールの更新設定
Homebrew 本体や管理下のツールの更新タイミングを設定できます。

| 設定項目 | タイミング | 対象 | デフォルト |
|---|---|---|---|
| `homebrew.global.autoUpdate` | `brew` コマンド 手動実行時 <br> （`brew install` 等） | 本体 + パッケージ | `true` |
| `homebrew.onActivation.autoUpdate` | `darwin-rebuild switch` | 本体 + パッケージ | `false` |
| `homebrew.onActivation.upgrade` | `darwin-rebuild switch` | パッケージのみ | `false` |



デフォルト値だと、従来の Homebrew の運用のままとなります。
更新のために `brew update`、`brew upgrade` を手動実行してください。

Nix のパッケージ更新管理に似た運用とする場合、以下のように設定します。

```nix:~/work/dotfiles/configuration.nix
  homebrew = {
    global.autoUpdate = false;
    onActivation = {
      upgrade = true;
      autoUpdate = false;
    };
  };
```

これにより、明示的にパッケージのソース情報（formulae）を更新した（`brew update`）後、環境を更新（`sudo darwin-rebuild switch`）することでパッケージを更新する、といった運用になります。


:::message
`nix flake update` しない限り、`sudo darwin-rebuild switch` してもパッケージのバージョンは変わりません。
これと同様に、上記設定だと `brew update` しない限り、パッケージのバージョンは変わりません。

ただし、Homebrew でのパッケージ管理なので、新しいパッケージを追加した際、依存となっているパッケージが意図せず自動的に更新されることはあると思います。
:::


```nix:~/work/dotfiles/configuration.nix
  homebrew = {
    onActivation = {
      cleanup = "uninstall";
      upgrade = true;
      autoUpdate = false;
      extraFlags = [
        "--verbose"
      ];
    };
    global.autoUpdate = false;
  };
```



# 4. 反映する
設定を反映します。

```bash:Bash
sudo darwin-rebuild switch
```
