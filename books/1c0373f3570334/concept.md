# コンセプト

# 誰のための本か（読者像）
- 想定職種/ロール：エンジニア全般、Homebrew で環境管理している Mac ユーザー
- 前提知識・経験：Mac の開発環境を自力で整えた経験がある
- 読者の課題/悩み：Nix に興味はあるが導入のハードルが高い、環境の再現性を高めたい
- 読了後に得たい状態：手軽にユーザー/開発環境を再現でき、Nix 運用を始められる

# 競合・類似書/記事との差別化

# 構成案（章立て）
1. Nix 導入の全体像（なぜ Nix か、最終形のゴール）
2. home-manager でユーザー環境を小さく始める
3. home-manager の限界と nix-darwin で補う（Homebrew の宣言管理）
4. flakes devshell で開発環境を分離・再現する

# 各章の狙い（簡潔に）
- 第1章：ユーザー環境/開発環境を Nix で一元管理する利点と、最終形の到達点を示す
- 第2章：home-manager 単体の導入と効果を実感し、Nix のメリットを掴む
- 第3章：home-manager で管理できないソフトを nix-darwin+Homebrew で宣言的に管理する
- 第4章：devshell で開発専用ツールを分離し、再現性と可搬性を高める

# 具体例・ケーススタディ案
- home-manager で CLI ツール群を管理し、macOS 新規セットアップを再現する
- home-manager では管理できない GUI アプリを nix-darwin+Homebrew で管理する
- devshell でプロジェクト専用の開発ツールを自動セットアップする
