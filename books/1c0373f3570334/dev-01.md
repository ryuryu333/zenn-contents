# 1. この章でやること
この章では、開発環境レイヤー（`dev-*`）の進め方を整理します。

**プロジェクトごとに開発環境を分離・再現するための導線**を示します。


# 2. 前提
このレイヤーは、共通レイヤー（`entrypoint-01`, `common-01` 〜 `common-04`）が完了している前提です。

:::message
`user` / `mac` レイヤーは必須ではありません。
ただし、普段のツール管理も Nix に寄せたい場合は先に `user` レイヤーを進めると運用が揃います。
:::


# 3. 読む章
以下の順で進めます。

1. `dev-02`: Flakes devShell で開発環境を管理
2. `dev-03`: 便利ツール（direnv / nix-direnv など）
3. `dev-04`: 個別ツールのバージョン管理


# 4. このレイヤーの完了条件
以下を満たしていれば、このレイヤーは完了です。

1. 開発プロジェクトで `devShell` を利用できる
2. プロジェクトに入ったときに必要ツールを再現できる
3. ツールのバージョン差分を管理できる


# 5. 次に進む判断
この先は用途で分岐します。

1. 開発環境の再現性が主目的だった: ここで止めて運用開始
2. Mac 全体も宣言管理したい: `mac-01` へ進む
