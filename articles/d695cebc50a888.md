# 実行手順
## 利用可能なディストリビューションの確認
以下のコマンドで確認できます。

今回は `Ubuntu` を利用する事とします。

```powershell:PowerShell
> wsl --list --online
インストールできる有効なディストリビューションの一覧を次に示します。
'wsl.exe --install <Distro>' を使用してインストールします。

NAME                            FRIENDLY NAME
AlmaLinux-8                     AlmaLinux OS 8
AlmaLinux-9                     AlmaLinux OS 9
# 省略...
Ubuntu                          Ubuntu
Ubuntu-22.04                    Ubuntu 22.04 LTS
```

## 任意の環境名を指定してインストール
`wsl --install` の `--name` オプションで任意の環境名を指定できます。

```powershell:PowerShell
wsl --install Ubuntu --name MyEnv
```

#### インストールの確認
インストール済みのディストリビューション一覧を確認すると、`MyEnv` という環境が追加されていることが確認できました。

```powershell:PowerShell
> wsl -l -v
  NAME              STATE           VERSION
* Ubuntu            Running         2
  docker-desktop    Running         2
  MyEnv            Stopped         2
```

## 環境への入り方
以下のコマンドで特定の環境へ入ることができます。

```powershell:PowerShell
wsl --distribution MyEnv
```

## 規定の環境を変更する
`wsl` コマンドをオプション無しで実行した場合、規定の環境へ入ります。

例えば、以下の場合だと `*` が記載されている `Ubuntu` が規定となっています。

```powershell:PowerShell
> wsl -l -v
  NAME              STATE           VERSION
* Ubuntu            Running         2
  docker-desktop    Running         2
  MyEnv            Stopped         2

# これを実行すると Ubuntu に入る
> wsl
```

以下のコマンドで規定の環境を `MyEnv` に変更できます。

```powershell:PowerShell
wsl --set-default MyEnv
```

```powershell:PowerShell
> wsl -l -v
  NAME              STATE           VERSION
* MyEnv             Running         2
  docker-desktop    Running         2
  Ubuntu            Running         2
```


# 上記方法が実行できない場合の別解
## エラー例
例えば、`Ubuntu-22.04` をインストールしようとすると、以下の様なエラーが発生します。
特定のディストリビューションでは `--name` が使えないようです。

```powershell:PowerShell
> wsl --install Ubuntu-22.04 --name MyEnv
レガシ ディストリビューションをインストールする場合、'--name' はサポートされません。
```

## エクスポート・インポート機能を利用する
WSL の既存環境をエクスポートし、新しい環境としてインポートする（＝環境を複製する）方法を用います。

### 複製元の環境の準備
`--name` オプションを利用せずに `Ubuntu-22.04` をインストールします。

```powershell:PowerShell
wsl --install Ubuntu-22.04
```

### 環境の複製
以下を実行します。

```powershell:PowerShell
wsl --export Ubuntu-22.04 - | wsl --import MyEnv "C:\Users\MyName\Wsl\MyEnv" -
```

#### コマンドの解説
```powershell
wsl --export <Distribution Name> <FileName>
wsl --import <Distribution Name> <InstallLocation> <FileName>
```

本来、`export` コマンドでは `.tar` ファイルとして環境データを保存します。

今回は複製が目的なので、ファイルを残さずに `import` コマンドに渡します。
そのため、`export` と `import` を `|` で連結し、`<FileName>` には標準出力/入力を意味する `-` を指定しています。

`<Distribution Name>` には `wsl -l -v` で表示された環境名を指定します。
`<InstallLocation>` には任意のフォルダを指定します。


:::details データのバックアップを残したい場合
以下を実行すると、コマンドを実行したディレクトリに保存できます。

```
wsl --export Ubuntu-22.04 Backup.tar
```

:::

### 不要な環境の削除
最後の後始末として、不要となった複製元である `Ubuntu-22.04` を削除します。

```
wsl --unregister Ubuntu-22.04
```

本当は公式ドキュメント（Microsoft Learn）を提示したかったのですが、`wsl --install` コマンドのオプションを解説したページが見つけられなかったです。



```powershell:PowerShell
> wsl --install --help
    --install [Distro] [Options...]
        Linux 用 Windows サブシステム ディストリビューションをインストールします。
        有効なディストリビューションの一覧を表示するには、'wsl.exe --list --online' を使用します。

        オプション:
            --enable-wsl1

            --fixed-vhd
                ディストリビューションを保存するための固定サイズのディスクを作成します。

            --from-file <Path>
```
