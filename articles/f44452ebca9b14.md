:::details SSH公開鍵認証 のセッティング
後ほどメイン PC から Raspberry Pi 5 をリモート操作する為に SSH を有効にしています。
セキュリティを確保する為、公開鍵認証のみを選択しています。

```powershell:PowerShell
# フォルダが無い場合は作成してください
cd ~/.ssh
# ed25519形式で鍵を作成
# passphrase（任意の文字列）の入力を要求されます
# Enter = 空欄 も可能ですがセキュリティを考慮すると付けた方が無難です
ssh-keygen -t ed25519 -f id_ed25519_raspberry_pi_5
# 公開鍵を表示
cat id_ed25519_raspberry_pi_5.pub
# 出力例
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPqbfNrhelNNexf/kQkSxoxLDlpHOdDcrOk7e8n6vqe4 <your_username>@main
```
出力した公開鍵をサービスタブの設定画面に貼り付けます。

続いて、今後のために追加の設定を行います。
SSH接続をする際、本来のコマンドは以下の通りです。
```powershell:PowerShell
ssh <ユーザ名>@<RaspberryPiのIP> -i <秘密鍵のパス>
```
毎回これを入力するのは大変ですので、簡便化の設定をします。
```powershell:PowerShell
# configをVSCodeで開く
# 無い場合は作成してください
code ~/.ssh/config
```
以下の様に記載します。
```:~/.ssh/config
Host raspberrypi.local 
    HostName raspberrypi.local # ラズパイのローカルホスト名
    IdentityFile ~/.ssh/id_ed25519_raspberry_pi_5 # ラズパイ用SSH鍵
    User pi # ラズパイのユーザー名
    IdentitiesOnly yes # 指定した鍵のみ使用
```
これで今後は以下のコマンドで Raspberry Pi 5 に接続できるようになりました。
```powershell:PowerShell
ssh raspberrypi.local
```

Raspberry Pi の表面に電源用の USB Type-C ポートがあるので、ACアダプターを接続します。
LEDランプが点灯します。最初はチカチカと点滅しますが、数分待つと緑色の点灯状態で安定します。

### 2.2.2 インストール用のパッケージの準備
インストールする OS やアーキテクチャを指定して NVIDIA のレポジトリ情報を登録します。
その後、パッケージ情報を取得します。

```powershell:PowerShell
# NVIDIAリポジトリのキーリングをダウンロード
<your_name>@raspberrypi:~$ wget https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/sbsa/cuda-keyring_1.1-1_all.deb
# キーリングをシステムに登録
<your_name>@raspberrypi:~$ sudo dpkg -i cuda-keyring_1.1-1_all.deb
# パッケージ情報を更新
<your_name>@raspberrypi:~$ sudo apt-get update
```

:::details Unable to locate package cuda-toolkit-12-8 となった場合
私の環境だと上記コマンドでは NVIDIA リポジトリが正しく設定されていない状態になりました。

```powershell:PowerShell
deb [signed-by=/usr/share/keyrings/cuda-archive-keyring.gpg] https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/sbsa/ /
EOF
deb [signed-by=/usr/share/keyrings/cuda-archive-keyring.gpg] https://developer.download.nvidia.com/compute/cuda/repos/ubuntu2404/sbsa/ /
```
