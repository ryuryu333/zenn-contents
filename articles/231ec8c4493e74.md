# Node.js のインストール
## バージョン管理ツール nvm-windows のインストール
Gemini CLI 用の環境として管理したいので、バージョン管理ツールを導入します。

https://github.com/coreybutler/nvm-windows

### 既存環境の確認・クリーンアップ
公式の Docs に従い、既存の Node.js があるかをチェック、存在するならがアンインストール。

- 以下のコマンドで Ver 名が出たらインストール済み

```powershell:PowerShell
node --version
```

- `%ProgramFiles%\nodejs` が存在するかチェック、あるなら消す

### インストーラーの実行
releases から `nvm-setup.exe` をダウンロード。
実行する。

https://github.com/coreybutler/nvm-windows/releases

・参考
https://qiita.com/akipon0821/items/eaeffe79221cfcd4d258

・インストールの確認

```powershell:PowerShell
> nvm version
1.2.2
```

## インストール
以下を順に実行。

```powershell:PowerShell
# Node.js のインスール
>nvm install 22.17.0

# 使用するバージョンを選択
> nvm use 22.17.0
Now using node v22.17.0 (64-bit)

# 現在の Node.js のバージョンを確認
> nvm current
v22.17.0
```

# VSCode の拡張 Gemini Code Assist の導入
このセッションの手順は以下を参考にしています。

## インストール
Gemini CLI を VSCode で良い感じに使えそうな拡張を google が出しているので、これを利用します。

>Gemini Code Assist は、オープンソースの AI エージェントである Gemini CLI へのアクセスを提供します。

https://codeassist.google/

```

```

![拡張選択画面](/images/231ec8c4493e74/231ec8c4493e74-2025-6-26_1.webp)

## Google アカウントへのログイン
VSCode のタブから `Gemini Code Assist` を選択し、Sign in のボタンを押す。
自身の Goole アカウントを選択してログインする。

>私は Gemini Diffusion の使用申し込みに使ったアカウントを使いました。特に有料プランに入っていませんが、問題なく使えました。有難い。 

![Gemini Code Assist の画面](/images/231ec8c4493e74/231ec8c4493e74-2025-6-26_2.webp)

ログイン後は3分ほど待っても以下の画面から変わりませんでした。
VSCode を一度落として立ち上げなおすと、使用可能な状態になっていました。

![ログイン後の画面](/images/231ec8c4493e74/231ec8c4493e74-2025-6-26_3.webp)

## Agent モードの有効化
Gemini Code Assist の設定を変更して Insiders ビルドを指定します。

`Ctrl` + `,` を押して VSCode の設定画面を開き、以下の値に設定します。

```
"geminicodeassist.updateChannel": "Insiders",
```

![設定例](/images/231ec8c4493e74/231ec8c4493e74-2025-6-26_6.webp)

VSCode を再起動します。

Gemini Code Assist タブを開くと、先ほどまで無かった Agent トグルが出現しました。

![Agentモード有効化の確認](/images/231ec8c4493e74/231ec8c4493e74-2025-6-26_7.webp)

# Agent コンテキストファイル（GEMINI.md）の設定
Agent の制御を行うファイル `GEMINI.md` を作成します。
（Claude Code における `CLAUDE.md`、Github Copilot における `copilot-instructions.md`）

以下の3パターンの配置場所があります。

| Scope                                                   | Location                                                                                                 |
|---------------------------------------------------------|----------------------------------------------------------------------------------------------------------|
| All your projects                                       | `~/.gemini/GEMINI.md`                                                                                    |
| A specific project                                      | 	Your working directory or any parent directories up to either your project root (identified by a .`git` folder) or your home directory. |
| A specific component, module, or sub-section of a project | Subdirectories of your working directory.                                                                 |

>公式ドキュメントより引用

https://developers.google.com/gemini-code-assist/docs/use-agentic-chat-pair-programmer#create-context-file

``

`GEMINI.md` での具体的な指示ですが、公式に良さげな例が載っていました。
これを参考に作ると良いでしょう。
