# はじめに
GitHub 連携によるバージョン管理や VSCode での記事作成に魅力を感じたので、セッティングしてみました。

普通に設定を進めると、Node.js や Zenn CLI をインストールしたり、VSCode（＋フォーマッター等の拡張機能）の設定を Zenn 仕様にしたり、と手間がかかります。
また、可能ならローカル環境はクリーンに保ちたいです。

そこで、Dev Containers を用いて Zenn 記事作成用環境を構築してみました。

主なメリットは以下の通りです。

- アプリの依存関係をコンテナ内に隔離し、ローカル環境をクリーンに保てる
- VSCode の設定や Markdown 関連の拡張機能を Zenn 執筆専用の環境として分離できる
- 環境構築の手順をコード (Dockerfile, devcontainer.json 等) で管理できる
- 異なるマシンでも同じ執筆環境を再現できる

## Dev Containers 設定ファイル作成
ローカルレポジトリ（筆者例:zenn_contents）直下に .devcontainer フォルダを作成します。
下記を参考に、.devcontainer フォルダ内に Dev Containers 用の設定ファイルを作成します。

```:フォルダ構成
zenn_contents/
├── .git/
└── .devcontainer/
    └── devcontainer.json
    ├── docker-compose.yml
    └── Dockerfile
```

### devcontainer.json
どの docker-compose.yml を使用するかを指定します。
VSCode で使用する拡張機能や設定を指定します。

```json:devcontainer.json
{
    "name": "zenn_contents",
    "dockerComposeFile": "docker-compose.yml",
    "service": "zenn_contents",
    //VScode接続時に開くフォルダを指定
    "workspaceFolder": "/work/zenn_contents",
    //VScodeの環境を指定
    "customizations": {
        "vscode": {
            //コンテナ作成時にインストールする拡張機能を記載
            "extensions": [
                "GitHub.copilot",
                "GitHub.copilot-chat",
                "negokaz.zenn-editor"
            ],
            //VScodeの設定を記載
            "settings": {
                //mdファイルに画像を貼り付けた際の保存先をzenn仕様に設定
                //https://zenn.dev/zenn/articles/deploy-github-images
                //https://code.visualstudio.com/docs/languages/markdown#_inserting-images-and-links-to-files
                "markdown.copyFiles.destination": {
                    "/articles/**/*": "images/articles/${documentBaseName}/",
                    "/books/**/*": "images/books/${documentBaseName}/"
                }
            }
        }
    }
}
```

### docker-compose.yml
ここと Dockerfile は普段の Docker と同じ書き方となります。
実は、フォルダのコンテナ:ローカル間共有は devcontainer.json でも設定できます。
個人的な好みで、Docker 側で設定できる箇所は Docker で行い、docker-compose.yml には VSCode 関連のみを記載する方針にしています。


```docker:docker-compose.yml
services:
  zenn_contents:
    build:
      dockerfile: Dockerfile
    tty: true
    # zenn用のフォルダ全てをマウント
    volumes:
      - ../:/work/zenn_contents/
```

### Dockerfile

```docker:Dockerfile
# 軽量なalpineイメージを選択
FROM node:22-alpine3.20
ENV NODE_PATH /opt/node_modules
# gitをコンテナ内で使用する際、ssh接続が必要なため、openssh-clientをインストール
RUN apk update && \
    apk add --no-cache git bash openssh-client
# zenn-cliのインストール
WORKDIR /work
RUN npm init --yes && \
    npm install -g zenn-cli@latest
# 実際に記事を管理するフォルダを作成
# コンテナとホスト間で共有するフォルダ
WORKDIR /work/zenn_contents
```

## SSH の設定
Zenn 記事の作成から公開まで一貫してコンテナ内で実施できた方が便利だと思います。
GitHub と SSH 接続にて通信している場合、コンテナ内で git push コマンドを使用するには、別途設定が必要となります。

https://code.visualstudio.com/remote/advancedcontainers/sharing-git-credentials#_using-ssh-keys

現状だと、コンテナ内で git push すると SSH 接続に用いる秘密鍵の情報がないとエラーになります。
Dev Containers ではローカルで実行中の ssh-agent が存在すれば、SSH 認証情報をコンテナ内で使用できる機能があり、こちらを利用します。

ssh-agent が自動起動する様に設定します。

```powershell:PowerShell
# SSHエージェントサービスを自動起動に設定
Set-Service ssh-agent -StartupType Automatic
# SSHエージェントサービスを開始
Start-Service ssh-agent
```

もしも、ssh-agent に GitHub 用の秘密鍵を登録していない場合、登録します。

```powershell:PowerShell
# .ssh直下の秘密鍵を登録、e.g. ~/.ssh/id_ed25519
ssh-add
# 特定の秘密鍵を登録
ssh-add ~/.ssh/GitHub/id_ed25519
```
