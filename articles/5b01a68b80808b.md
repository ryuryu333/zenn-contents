# はじめに
VSCode と Zenn CLI を使って記事を書いていると、Markdown の体裁・日本語表現・リンク切れ・スペルなど、複数の観点をまとめてチェックしたくなります。

本記事では、lint ツールを個別に叩くのではなく、`treefmt` をハブにして一括実行できるよう整備した内容を紹介します。設定はすべてリポジトリ内の `linter/` 配下にまとめ、`task` コマンドひとつでチェックできます。


# 想定読者
- Zenn CLI／VSCode で執筆している方
- Markdownlint や textlint を使っている（使いたい）方
- 複数のリンターを統一手順で回したい方


# 概要
- `treefmt` に各リンターを登録し、`task` で一括実行
- 対象は `articles/*.md` に限定（既存記事は段階的に移行）
- 設定は `linter/` に集約し、理由を明文化
- Nix の devShell にツールを同梱し、セットアップ不要で実行


# フォルダ構成
```:構成
zenn_contents/
├─ articles/
│  └─ <article>.md
├─ linter/
│  ├─ treefmt.toml
│  ├─ .markdownlint-cli2.jsonc
│  ├─ .textlintrc.json
│  └─ cspell.json
├─ Taskfile.yml
├─ task/
│  └─ utl.yml
└─ flake.nix
```


# 使用ツールと目的
- markdownlint-cli2: Markdown の表記・構文のスタイルチェック
- textlint: 日本語の表記・文体ルールのチェック（技術文書向け）
- lychee: URL のリンク切れ検知（タイムアウトとリトライを設定）
- cspell: スペルチェック（用語辞書をプロジェクトに同梱）


# クイックスタート（既に Nix 環境がある前提）
最短で試す場合は、リポジトリ直下で以下を実行します。

```bash
# devShell を起動
nix develop

# すべてのリンターを一括実行（既定タスク）
task
# もしくは明示的に
# task lint

# 直接 treefmt を叩くことも可能
treefmt --config-file ./linter/treefmt.toml
```

:::message
`task` は `Taskfile.yml` で `treefmt` を呼び出しています。
- `Taskfile.yml` の既定タスクは `utl:lint`
- `task/utl.yml` で `treefmt --config-file ./linter/treefmt.toml` を実行
:::


# treefmt の設定
`linter/treefmt.toml` に各リンターを登録し、実行順と対象ファイルを定義しています。

```toml:linter/treefmt.toml
[formatter.md]
command = "markdownlint-cli2"
options = ["--config", "linter/.markdownlint-cli2.jsonc"]
includes = ["articles/*.md"]
priority = 1

[formatter.text]
command = "textlint"
options = ["--config", "linter/.textlintrc.json"]
includes = ["articles/*.md"]
priority = 2

[formatter.url]
command = "lychee"
options = ["--timeout", "10", "--max-retries", "2"]
includes = ["articles/*.md"]
priority = 3

[formatter.spell]
command = "cspell"
includes = ["articles/*.md"]
priority = 4

[global]
excludes = [
  # 既存の記事は段階的に対応するため一時的に除外
]
```

- 実行順序は「体裁 → 文体 → リンク → スペル」の順に設定しました。
- 既存記事はスタイルが固まるまで `excludes` で除外し、新規記事から適用しています。


# 各リンターの設定と方針
## markdownlint-cli2
`linter/.markdownlint-cli2.jsonc` でルールを調整しています。

```json:linter/.markdownlint-cli2.jsonc
{
  "config": {
    "MD001": false,
    "MD007": false,
    "MD012": false,
    "MD013": false,
    "MD022": false,
    "MD024": false,
    "MD025": false,
    "MD029": false,
    "MD033": false,
    "MD034": false
 }
}
```

主な無効化理由（抜粋）
- MD013: 1 行の文字数制限を撤廃（原文の可読性を優先）
- MD025: 最上位見出しの扱い（Zenn はフロントマターでタイトル管理）
- MD033: インライン HTML 許容（必要に応じて柔軟に記法を使う）
- MD034: ベア URL を許容（Zenn でのリンク提示を簡潔に）
- そのほかは、既存記事との整合と執筆フローの簡素化を優先

### 例（Before/After）
MD013（行長）
```markdown:Before
これは非常に長い説明文ですが、折り返しを強制しなくても文脈上は読みやすいので、1 行のままにしておきます。Zenn のプレビューや表示幅次第で適切に折り返されます。
```

```markdown:After（許容）
これは非常に長い説明文ですが、折り返しを強制しなくても文脈上は読みやすいので、1 行のままにしておきます。Zenn のプレビューや表示幅次第で適切に折り返されます。
```

MD025（最上位見出し）
```markdown:Before
# タイトル（フロントマターと重複）
```

```markdown:After
## はじめに
```

MD033（インライン HTML）
```markdown:Before
テーブルの直後に強制改行が必要な場合があります。<br>
```

```markdown:After（許容）
テーブルの直後に強制改行が必要な場合があります。<br>
```

MD034（ベア URL）
```markdown:Before
<https://example.com>
```

```markdown:After（許容）
https://example.com
```


## textlint
`linter/.textlintrc.json` では、日本語の文体と表記ルールを中心に調整しています。

```json:linter/.textlintrc.json
{
  "filters": { "comments": true },
  "rules": {
    "preset-ja-spacing": {
      "ja-space-between-half-and-full-width": { "space": "always" }
    },
    "preset-ja-technical-writing": {
      "ja-no-weak-phrase": false,
      "no-mix-dearu-desumasu": { "preferInList": "ですます" },
      "ja-no-mixed-period": { "allowPeriodMarks": [":"] },
      "no-exclamation-question-mark": false
    },
    "terminology": {}
  }
}
```

- 半角・全角の間にスペースを入れる: 記号や英単語と日本語の混在を読みやすくするため
- 文体は「です・ます」を基準に統一: 読み味の揺れを抑制
- 弱い表現や「！？」などは許容: 技術記事で必要なニュアンスを残すため
- `terminology`: 用語のゆれ検知を将来拡張しやすいよう導入（最小構成）
- `filters.comments`: 必要に応じて `<!-- textlint-disable -->` で個別除外が可能

### 例（Before/After）
半角・全角の間にスペースを入れる（`preset-ja-spacing`）
```markdown:Before
Zenn執筆用の環境を整えます。
```

```markdown:After
Zenn 執筆用の環境を整えます。
```

文体の統一（です・ます、`no-mix-dearu-desumasu`）
```markdown:Before
この方法は簡単です。設定は次のとおりである。
```

```markdown:After
この方法は簡単です。設定は次のとおりです。
```

記号の許容（`ja-no-mixed-period` の `:` 許可、`no-exclamation-question-mark` 無効）
```markdown:Before
注意: 設定ファイルはリポジトリに保存します。便利です！
```

```markdown:After（許容）
注意: 設定ファイルはリポジトリに保存します。便利です！
```


## cspell
`linter/cspell.json` に対象と辞書を記載しています。

```json:linter/cspell.json
{
  "version": "0.2",
  "files": ["articles/*.md"],
  "words": ["treefmt", "Zenn"]
}
```

- 固有名詞やツール名は辞書に追加し、誤検知を軽減
- ファイル対象は `articles/*.md` に限定


## lychee（リンク切れチェック）
- オプション: `--timeout 10`、`--max-retries 2`
  - フレーク対策としてタイムアウト短め＋少回数のリトライ
- 一時的に落ちているサイトでアラートが出た場合は、再実行や除外リストの導入を検討


# 運用のポイント
- 新規記事から順次適用し、既存記事は `excludes` を外しながら段階的に整備
- どうしても引っかかる箇所は、ルールの根拠を見直すか、ピンポイントで無効化
- 実行は `task` に集約し、手元で同じコマンドを呼ぶ

## `excludes` の外し方（段階移行）
`linter/treefmt.toml` の `[global].excludes` に既存記事を列挙しています。対応した記事から順にエントリを削除し、`task` で通ることを確認します。

```toml:linter/treefmt.toml（抜粋）
[global]
excludes = [
  "articles/00341ed49b7935.md",
  # ... 対応できたら削除
]
```

## 辞書・除外の更新
- cspell の辞書は `linter/cspell.json` の `words` に追記します（固有名詞やツール名など）。
- textlint は一時的に無視したい箇所にコメントを使えます。

```markdown
<!-- textlint-disable -->
この段落は一時的に除外します。
<!-- textlint-enable -->
```

必要に応じて、ルール側の設定（`.markdownlint-cli2.jsonc` / `.textlintrc.json`）を見直して恒久対応します。


# おわりに
`treefmt` をハブにすることで、Markdown・日本語・リンク・スペルのチェックを一本化できました。設定をリポジトリに同梱しておくと、執筆端末が変わっても同じ品質基準で書き進められます。必要に応じてルールの粒度を調整し、運用しやすい形に育てていきます。
