# はじめに
本記事では **nix-darwin 単独で Caps を Fn にマッピングし、入力ソース切替に使う方法**を紹介します。  

切替には nix-darwin の `system.keyboard.userKeyMapping` を利用します。

現時点（2025/2/1）では、この機能は[公式リファレンス](https://nix-darwin.github.io/nix-darwin/manual/)に記述されていません[^1]。
そのため、本記事では **`system.keyboard.userKeyMapping` の利用方法も併せて解説します**。

[^1]:`system.keyboard.enableKeyMapping` や `system.keyboard.remapCapsLockToControl` 等は[公式リファレンス](https://nix-darwin.github.io/nix-darwin/manual/)に記述されています。
一方、`system.keyboard.userKeyMapping` は検索をかけてもヒットしません。
[ソースコード（nix-darwin/modules/system/keyboard.nix）](https://github.com/nix-darwin/nix-darwin/blob/master/modules/system/keyboard.nix)を確認すると、`system.keyboard.userKeyMapping` が定義されています。

# 概要
以下のような設定を記述します。

最初に、マッピングしたいキーの ID を[ここから](https://developer.apple.com/library/archive/technotes/tn2450/_index.html)調べ、16 進数から 10 進数に変換してください。
その後、`(remap <置換元のキー ID> <置換先のキー ID>)` と記載してください。

`sudo darwin-rebuild switch` すればマッピングが反映されます。

```nix:configuration.nix
{
  pkgs,
  ...
}:
{
  system.keyboard = {
    enableKeyMapping = true;
    userKeyMapping =
      let
        remap = src: dst: {
          HIDKeyboardModifierMappingSrc = src;
          HIDKeyboardModifierMappingDst = dst;
        };
        caps_lock = 30064771129; # 0x39
        fn_key = 1095216660483; # 0xFF00000003
      in
      [
        # Caps Lock -> Fn
        (remap caps_lock fn_key)
      ];
  };
}
```

# 1. 課題（Caps で日本語/英語切替が不安定）
私は普段、macOS の「**Fn キーで入力ソースを切り替える**」設定を使っています。
しかし Caps を入力ソース切替に割り当てると、以下のような不満がありました。

- **日本語入力直後に Caps を押しても反応しないことがある**
- **普通に押したつもりが長押し判定になり、Caps Lock が有効化される**

この不安定さが気になり、**Caps を Fn に置き換える方針**にしました。


:::details Caps について
Apple コミュニティを調べてみると、反応性について同じような投稿がありました。
環境起因の問題なのか、Mac の仕様なのか不明ですが、少なくとも私にとって不便であり、かつ、キーマッピングを切り替えたら解消したのは確かです。

https://discussions.apple.com/thread/250797717?sortBy=rank

https://discussionsjapan.apple.com/thread/254890499?sortBy=rank

以下の公式の解説を読む限り、反応が悪いと感じるのは Mac の仕様なのかもしれません。

>メディアイジェクトキーまたはCaps Lockキーが機能しない場合
キーを長く押し続けてみてください。これらのキーは、誤って押されてもすぐに反応しないよう、少し遅れて機能するようになっています。画面にメディアイジェクトアイコンが表示されるまでメディアイジェクトキーを押したままにしてください。キーのインジケータランプが点灯するまでCaps Lockキーを押したままにしてください。

https://support.apple.com/ja-jp/guide/mac-help/mchlp1240/mac

:::


# 2. Caps を Fn（Globe）に変更する
キーボード設定によりますが、デフォルトでは Fn キーを押すと入力ソースが変更されるかと思います。

このキーならば反応性に問題無いはずですが、キーの配置場所が押しにくかったり、外付けキーボードにはそもそも搭載されていないことも珍しくありません。

そこで、押しやすい位置にある Caps キーを Fn キーに変更することにしました。


# 3. 設定手順
## 3.1 キー ID の確認
どのキーを入れ替えるかを指定するために、「どのデバイス」の「どのキー」であるか表すコードが必要です。

マッピングを変えたいキーの ID を[この資料の Key Table Usages](https://developer.apple.com/library/archive/technotes/tn2450/_index.html) から調べます。

![Key Table の例](/images/a6fc32a05be6d0/image.png)

Caps Lock キーの場合、0x39 です。
この場合、キー ID は 0x700000039 となります。

他のキーの場合、末尾 2 桁を変えてください。

求めた値は 16 進数になっているので、適当な方法で 10 進数に変換してください。
Caps Lock キーの場合、30064771129 となります。


:::details コードの詳細
話が脱線しますが、詳細が気になる人用です。

- HID Usage Tables 1.7

https://www.usb.org/document-library/hid-usage-tables-17

HID Usage Tables p17 の定義より、キーボードを意味する Page ID は 0x07 です。

![Page ID 一覧](/images/a6fc32a05be6d0/image1.png)

HID Usage Tables p89 の定義より、キーボードにおける Caps Lock キーを意味する Usage ID は 0x39 です。

![Usage ID 一覧](/images/a6fc32a05be6d0/image2.png)

0x07000（キーボード）と 0x00039（Caps Lock キー）を組み合わせ（0x0700000039）、先頭の 0 を除いた値が利用するコードです（0x700000039）。
:::

Fn（Globe）キーは Apple 独自のキーなので、求め方が異なります。
特殊な例なので、0xFF00000003 = 1095216660483 とだけ知っていれば大丈夫です。

:::details Fn（Globe）キーについて
Apple 独自のキーのため、Page ID と Usage ID が特殊です。

Page ID は HID Usage Tables p17 の表の FF00-FFFF（Vendor-defined）が該当します。

![Page ID 一覧](/images/a6fc32a05be6d0/image3.png)

Usage ID は一次ソースは見つかりませんでした。

調べたところ、0xFF00000003 が left Fn キー、0xFF0100000003 が right Fn キーを表しているようです。

>right Fn キーは所有キーボードに無かったので未検証です。

https://apple.stackexchange.com/questions/340607/what-is-the-hex-id-for-fn-key%EF%BC%89

:::


## 3.2 nix-darwin 設定ファイルの作成
`system.keyboard.enableKeyMapping = true;` でキーマッピング機能を有効化します。

`system.keyboard.userKeyMapping` にて具体的なキーマッピングを指定します。

```nix:configuration.nix
{
  pkgs,
  ...
}:
{
  system.keyboard = {
    enableKeyMapping = true;
    userKeyMapping = [
      {
        # Caps Lock キー -> Fn（Globe）キー
        HIDKeyboardModifierMappingSrc = 1095216660483; # Caps Lock キー
        HIDKeyboardModifierMappingDst = 30064771129; # Fn（Globe）キー
      }
    ];
  };
}
```

## 3.3 反映コマンド
いつもの環境更新コマンドで反映させます。

```bash:terminal
sudo darwin-rebuild switch
```


# 4. 記述を楽にする
ここからはおまけです。
上記では可読性が悪いので、私は以下のように書いています。

```nix:configuration.nix
{
  pkgs,
  ...
}:
{
  system.keyboard = {
    enableKeyMapping = true;
    userKeyMapping =
      let
        remap = 
          let
            to_int = pkgs.lib.trivial.fromHexString;
          in
          src: dst: {
          HIDKeyboardModifierMappingSrc = to_int src;
          HIDKeyboardModifierMappingDst = to_int dst;
        };
        caps_lock = "700000039";
        fn_key = "FF00000003";
      in
      [
        # Caps Lock -> Fn
        (remap caps_lock fn_key)
      ];
  };
}
```

Caps Lock キーに加えて、私は Windows ライクの設定にするために Ctrl と command を入れ替えています。

コードにすると、以下のようになります。

```nix:configuration.nix
{
  pkgs,
  ...
}:
{
  system.keyboard = {
    enableKeyMapping = true;
    userKeyMapping =
      let
        remap = 
          let
            to_int = pkgs.lib.trivial.fromHexString;
          in
          src: dst: {
          HIDKeyboardModifierMappingSrc = to_int src;
          HIDKeyboardModifierMappingDst = to_int dst;
        };
        # Key-map References:
        #   https://developer.apple.com/library/archive/technotes/tn2450/_index.html
        # e.g.
        #   07000 = Keyboard, 000E0 = Keyboard Left Control
        #     -> 0x7000000E3 = Keyboard Left Command
        # macOS Fn key:
        #   https://apple.stackexchange.com/questions/340607/what-is-the-hex-id-for-fn-key%EF%BC%89
        left_control = "7000000E0";
        left_command = "7000000E3";
        caps_lock = "700000039";
        fn_key = "FF00000003";
      in
      [
        # Left Control <-> GUI(Command)
        (remap left_control left_command)
        (remap left_command left_control)
        # Caps Lock -> Fn
        (remap caps_lock fn_key)
      ];
  };
}
```
