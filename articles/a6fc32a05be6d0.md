# はじめに
本記事では **nix-darwin 単独で Caps を Fn にマッピングし、入力ソース切替に使う方法**を紹介します。  

切替には nix-darwin の `system.keyboard.userKeyMapping` を利用します。

現時点（2026/2/1）では、この機能は[公式リファレンス](https://nix-darwin.github.io/nix-darwin/manual/)に記述されていません[^1]。
そのため、本記事では **`system.keyboard.userKeyMapping` の利用方法も併せて解説します**。

[^1]:`system.keyboard.enableKeyMapping` や `system.keyboard.remapCapsLockToControl` 等は[公式リファレンス](https://nix-darwin.github.io/nix-darwin/manual/)に記述されています。
一方、`system.keyboard.userKeyMapping` は検索をかけてもヒットしません。
[ソースコード（nix-darwin/modules/system/keyboard.nix）](https://github.com/nix-darwin/nix-darwin/blob/master/modules/system/keyboard.nix)を確認すると、`system.keyboard.userKeyMapping` が定義されています。


:::message
`system.keyboard.userKeyMapping` は `internal = true;` と定義されています。
**ユーザーが直接利用することは想定されていないと思われます**。

ソースコードを確認の上、ご利用ください。
:::

# 概要
私が使っている設定を例として提示します。
`sudo darwin-rebuild switch` すればマッピングが反映され、Caps -> Fn へ変更されます。

```nix:configuration.nix
{
  pkgs,
  ...
}:
{
  system.keyboard = {
    enableKeyMapping = true;
    userKeyMapping =
      let
        mkKeyMapping =
          let
            hexToInt = s: pkgs.lib.trivial.fromHexString s;
          in
          src: dst: {
            HIDKeyboardModifierMappingSrc = hexToInt src;
            HIDKeyboardModifierMappingDst = hexToInt dst;
          };
        # Key-map References:
        #   https://developer.apple.com/library/archive/technotes/tn2450/_index.html
        # e.g.
        #   07000 = Keyboard, 000E3 = Left Command
        #     -> 0x7000000E3 = Keyboard Left Command
        # macOS Fn key:
        #   https://apple.stackexchange.com/questions/340607/what-is-the-hex-id-for-fn-key%EF%BC%89
        capsLock = "0x700000039";
        fnKey = "0xFF00000003";
      in
      [
        # Caps Lock -> Fn
        (mkKeyMapping capsLock fnKey)
      ];
  };
}
```

# 1. 課題（Caps で日本語/英語切替が不安定）
私は普段、Mac の「**Caps キーで入力ソースを切り替える**」設定を使っています。
しかし Caps を入力ソース切替に割り当てると、以下のような不満がありました。

- **日本語入力直後に Caps を押しても反応しないことがある**
- **普通に押したつもりが長押し判定になり、Caps Lock が有効化される**

:::details Caps の挙動について
Apple コミュニティを調べてみると、反応性について同じような投稿がありました。

https://discussions.apple.com/thread/250797717?sortBy=rank

https://discussionsjapan.apple.com/thread/254890499?sortBy=rank

以下の公式の解説を読む限り、反応が悪いと感じるのは Mac の仕様なのかもしれません。

>メディアイジェクトキーまたはCaps Lockキーが機能しない場合
キーを長く押し続けてみてください。これらのキーは、誤って押されてもすぐに反応しないよう、少し遅れて機能するようになっています。画面にメディアイジェクトアイコンが表示されるまでメディアイジェクトキーを押したままにしてください。キーのインジケータランプが点灯するまでCaps Lockキーを押したままにしてください。

https://support.apple.com/ja-jp/guide/mac-help/mchlp1240/mac

:::


# 2. Caps を Fn（Globe）に変更する
キーボード設定によりますが、デフォルトでは Fn キーを押すと入力ソースが変更されるかと思います。

Fn キーならば反応性に問題無いのですが、キーの配置場所が押しにくかったり、外付けキーボードでは未搭載であることも珍しくありません。

そこで、**押しやすい位置にある Caps キーを Fn キーへ変更**することにしました。


# 3. 設定手順
## 3.1 キー ID の確認
どのキーを入れ替えるかを指定するために、「どのデバイス」の「どのキー」であるかを表すコードが必要です。

マッピングを変えたいキーの ID を[この資料の Key Table Usages](https://developer.apple.com/library/archive/technotes/tn2450/_index.html) から調べます。

![Key Table の例](/images/a6fc32a05be6d0/image.png)

Caps Lock キーの場合、0x39 です。
**この場合、キー ID は 0x700000039 となります**。


:::message
**他のキーの場合、末尾 2 桁を変えてください**。

例えば、Command キーの場合、Keyboard Left GUI 0xE3 が該当します。
キー ID は 0x7000000E3 となります。
:::

求めた値は 16 進数になっています。
nix-darwin の `system.keyboard.userKeyMapping` では、10 進数にした値が必要となります。

**適当な方法で 10 進数に変換してください**。

例えば、Caps Lock キーの場合、0x700000039 = 30064771129 となります。

```zsh:Zsh
printf "%d\n" 0x700000039
```

:::message
**Fn（Globe）キーは Apple 独自のキーなので、求め方が異なります**。

特殊な例なので、Fn キー = 0xFF00000003 = 1095216660483 とだけ知っていれば大丈夫です。
:::


:::details キー ID の調べ方について
詳細が気になる人用です。

- HID Usage Tables 1.7

https://www.usb.org/document-library/hid-usage-tables-17

HID Usage Tables p17 の定義より、キーボードを意味する Page ID は 0x07 です。

![Page ID 一覧](/images/a6fc32a05be6d0/image1.png)

HID Usage Tables p89 の定義より、キーボードにおける Caps Lock キーを意味する Usage ID は 0x39 です。

![Usage ID 一覧](/images/a6fc32a05be6d0/image2.png)

0x07000（キーボード）と 0x00039（Caps Lock キー）を組み合わせ（0x0700000039）、先頭の 0 を除いた値が利用するコードです（0x700000039）。
:::


:::details Fn（Globe）キーについて
Apple 独自のキーのため、Page ID と Usage ID が特殊です。

Page ID は HID Usage Tables p17 の表の FF00-FFFF（Vendor-defined）が該当します。

![Page ID 一覧](/images/a6fc32a05be6d0/image3.png)

Usage ID について、一次ソースは見つかりませんでした。

調べたところ、0xFF00000003 が left Fn キー、0xFF0100000003 が right Fn キーを表しているようです。

>right Fn キーは所有キーボードに無かったので未検証です。

https://apple.stackexchange.com/questions/340607/what-is-the-hex-id-for-fn-key%EF%BC%89

:::


## 3.2 nix-darwin 設定ファイルの作成
`system.keyboard.enableKeyMapping = true;` でキーマッピング機能を有効化します。

`system.keyboard.userKeyMapping` にて具体的なキーマッピングを指定します。

```nix:configuration.nix
{
  pkgs,
  ...
}:
{
  system.keyboard = {
    enableKeyMapping = true;
    userKeyMapping = [
      {
        # Caps Lock キー -> Fn（Globe）キー
        HIDKeyboardModifierMappingSrc = 30064771129; # Caps Lock キー
        HIDKeyboardModifierMappingDst = 1095216660483; # Fn（Globe）キー
      }
    ];
  };
}
```

:::message
[system.keyboard.userKeyMapping](https://github.com/nix-darwin/nix-darwin/blob/master/modules/system/keyboard.nix) の description より、おそらく内部では `hidutil` を利用していると思われます（ソースコードは確認してません）。

>List of keyboard mappings to apply, for more information see <https://developer.apple.com/library/content/technotes/tn2450/_index.html>.

[Remapping Keys in macOS 10.12 Sierra](https://developer.apple.com/library/archive/technotes/tn2450/_index.html) では、以下のように `hidutil` コマンドでマッピングする例が記載されています。

```:コマンド例
hidutil property --set '{"UserKeyMapping":[{"HIDKeyboardModifierMappingSrc":0x700000004,"HIDKeyboardModifierMappingDst":0x700000005},{"HIDKeyboardModifierMappingSrc":0x700000005,"HIDKeyboardModifierMappingDst":0x700000004}]}'
```

`system.keyboard.userKeyMapping` でも同じ書き方で指定します。
ただし、型定義は以下のように整数型となっています。

>type = types.listOf (types.attrsOf types.int)

Nix は 16 進数を表す型が無いので、10 進数（int）として表現することで確実性を担保しているのかもしれません。
（仮に文字列型にすると、何でも記入できるようなものですし...）
:::


## 3.3 反映コマンド
いつもの環境更新コマンドで反映させます。

```bash:terminal
sudo darwin-rebuild switch
```

:::message
一度反映させれば、再起動してもマッピングは維持されました。

しかし、**変更するキーによっては再起動でマッピングがリセットされたと報告されています**。
対策もあるようですので、詳細は [nix-darwin の issue](https://github.com/nix-darwin/nix-darwin/issues/905) をご確認ください。
:::


# 4. 記述を楽にする（おまけ）
上記では可読性が悪いので、私は以下のように書いています。

```nix:configuration.nix

{

            HIDKeyboardModifierMappingDst = hexToInt dst;
          };

        fnKey = "0xFF00000003";
      in
      [
        # Caps Lock -> Fn
        (mkKeyMapping capsLock fnKey)
      ];
  };
}
```

Caps Lock キーに加えて、私は Windows ライクの設定にするため、Ctrl と command を入れ替えています。

最終的なコードは以下の通りです。

```nix:configuration.nix
{
  pkgs,
  ...
}:
{
  system.keyboard = {
    enableKeyMapping = true;
    userKeyMapping =
      let
        mkKeyMapping =
          let
            hexToInt = s: pkgs.lib.trivial.fromHexString s;
          in
          src: dst: {
            HIDKeyboardModifierMappingSrc = hexToInt src;
            HIDKeyboardModifierMappingDst = hexToInt dst;
          };
        # Key-map References:
        #   https://developer.apple.com/library/archive/technotes/tn2450/_index.html
        # e.g.
        #   07000 = Keyboard, 000E3 = Left Command
        #     -> 0x7000000E3 = Keyboard Left Command
        # macOS Fn key:
        #   https://apple.stackexchange.com/questions/340607/what-is-the-hex-id-for-fn-key%EF%BC%89
        leftControl = "0x7000000E0";
        leftCommand = "0x7000000E3";
        capsLock = "0x700000039";
        fnKey = "0xFF00000003";
      in
      [
        # Left Control <-> GUI(Command)
        (mkKeyMapping leftControl leftCommand)
        (mkKeyMapping leftCommand leftControl)
        # Caps Lock -> Fn
        (mkKeyMapping capsLock fnKey)
      ];
  };
}
```
