# はじめに
ネット上には様々な Nix インストーラーが公開されています。

**インストーラーによっては Nix のディストリビューション（改良版）をインストールするモノも存在し、初見ではどれを利用すべきか迷うこともあります**。

本記事ではメジャーなインストーラーを 4 つ選択し、それぞれでインストールした Nix の挙動の違いを確認しました。
具体的には、**インストールされる Nix の種類、バージョン、Nix の初期設定（`nix.conf`）、`nix develop` の実行時間**、を比較しました。

どのインストーラーを選ぶかの判断材料になれば幸いです。

:::message
**本記事での検証について**。

インストーラーの使用時期、OS やマシンスペックによって結果は変動します。
ゆえに「**私の環境ではこうだった**」という前提でお読みください。
:::


# 検証環境

- Windows 11
  - CPU: AMD Ryzen 5 5600X（6 コア、12 スレッド）
  - RAM: 32 GB
- WSL2 Ubuntu 24.04.4 LTS
- 実施時期: 2026/2/13-2026/2/16


# 1. 検証環境の準備
WSL にて、各インストーラー用の専用環境を用意します。

----

Ubuntu-24.04 をインストールし、パッケージの更新と Git の設定をします。

```powershell:PowerShell
wsl --install Ubuntu-24.04 --name Tmp-Env
```

```bash:Bash
sudo apt update && sudo apt upgrade -y
sudo apt clean
git config --global user.name "dummy"
git config --global user.email "dummy@local.test"
```

```bash:Bash
$ grep PRETTY_NAME /etc/os-release
PRETTY_NAME="Ubuntu 24.04.4 LTS"
```

`exit` した後、環境を停止させます。

```powershell:PowerShell
wsl --terminate Tmp-Env
```

Ubuntu 環境を複製します。

```powershell:PowerShell
$wslPath = "$HOME\wsl\"
$envs = "1-Nix-Classic", "2-Nix-Modern", "3-Determinate", "4-Lix"

wsl --export Tmp-Env "$wslPath\Tmp-Env.tar"
foreach ($name in $envs) {
    wsl --import $name "$wslPath\$name" "$wslPath\Tmp-Env.tar"
}
```

```powershell:PowerShell
wsl --unregister Tmp-Env
rm "$wslPath\Tmp-Env.tar"
```

これ以降、各環境内で操作していきます。

```powershell:PowerShell
wsl -d 1-Nix-Classic
wsl -d 2-Nix-Modern
wsl -d 3-Determinate
wsl -d 4-Lix
```

## 2.4 Lix

>Lix is an independent variant of the Nix package manager, developed by a team of open-source volunteers, and maintained by and for a passionate community of users.

:::message
2025 年末から、[nix-darwin](https://github.com/nix-darwin/nix-darwin) にて Lix が推奨されるようになったので、名前を見かけたことがある方もいるかもしれません（[c.f. Commit eb05ef6](https://github.com/nix-darwin/nix-darwin/commit/eb05ef6d653f9ed4d1b3bacf0f5795b4d07d6b48)）[^2]。
:::


[^2]: 以前は、Determinate Systems が提供するインストーラーにて Nix と Determinate Nix を選択してインストールできました。
2026 年初旬、upstream Nix（＝ Nix 本家）が選択できなくなりました。
（c.f. [Dropping upstream Nix from Determinate Nix Installer](https://determinate.systems/blog/installer-dropping-upstream/)）
この変更に伴い、nix-darwin 側のドキュメントも変わったと思われます。

## 4.4 巨大ファイルを追加した場合
`nix develop` を実行する際、Git が追跡しているファイル全体が `nix/store` にコピーされます。

>When using the flakes and nix-command experimental features, a local directory within a Flake is always copied into the Nix store completely unless it is a Git repository!
c.f. [nix.dev > Working with local files > File sets > Warning](https://nix.dev/tutorials/working-with-local-files.html)

そのため、`flake.nix` が置かれているプロジェクトが巨大になると devShells 起動時間が長くなります。

:::message
Git 管理している場合、巨大ファイル（バイナリ等）は gitignore する、`flake.nix` を置くフォルダを分ける（ルートに置かない）といった対策があります。

Determinate Nix では [Lazy trees という独自機能](https://determinate.systems/blog/changelog-determinate-nix-352/)で対策しています。
なお、[本家 Nix 側へ Lazy trees 追加の PR](https://github.com/NixOS/nix/pull/13225) が作成されていますが、現時点（2026/2/14）では技術的な課題により未実装です。
:::

----

`flake.nix` を置いているディレクトリに巨大ダミーファイルを作成し、`nix develop` の実行時間を比較します。

# 5. ローカルビルドの比較
## 5.1 前準備
通常、パッケージはローカルビルドされず、nixpkgs のバイナリキャッシュ（[cache.nixos.org](https://cache.nixos.org/)）からダウンロードされます。

しかし、パッケージのライセンスによっては、バイナリキャッシュがアップロードされていません。
この場合、ローカルでビルドが実行されます。

ローカルビルド時の `nix develop` の違いを見てみます。

----

まず、環境をできるだけ揃えたいので、GC をかけてキャッシュを消しておきます。
新しいディレクトを用意し、terraform を用いる devShells を定義します。

```bash:Bash
nix-collect-garbage -d
mkdir ~/work1
cd ~/work1
cat <<EOF > flake.nix
```

# 6. パッケージを多く指定した devShells の比較
## 6.1 devShells 起動時間
インストーラーによって、max-jobs、cores、auto-optimise-store などの初期値が異なります。

max-jobs はいくつ並行して derivation を計算するか決めます。
cores は derivation の計算にいくつの CPU コアを利用するかを決めます。

https://nix.dev/manual/nix/2.24/advanced-topics/cores-vs-jobs

auto-optimise-store を有効にすると、`nix/store/` にて内容が同じ場合、ハードリンクを利用して実ファイルを重複して保持しないようになります。
結果として、ディスク容量が節約できます。

https://nixos.wiki/wiki/Storage_optimization

https://nix.dev/manual/nix/2.24/command-ref/conf-file

----

各条件の初期設定を提示します。

|                       | max-jobs | cores | auto-optimise-store |
|:---------------------:|:--------:|:-----:|:-------------------:|
| Nix Classic Installer |    1     |   0   |        false        |
| Nix Modern Installer  |    12    |   0   |        true         |
|    Determinate Nix    |    12    |   0   |        false        |
|          Lix          |    12    |  12   |        true         |

>※ 0 の場合、利用可能な全てのコアを利用します。

----

# おわりに
インストーラーによって Nix ディストリビューションや初期設定（`nix.conf`）が異なり、コマンドの実行時間にも差が生じると分かりました。

評価時期によって、各ディストリビューションの性能は変わると思いますので、定期的な情報収集と検証が重要となります。
