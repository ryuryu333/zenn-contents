# はじめに
関数型まつり 2026 にて「パッケージマネージャー Nix はなぜ純粋関数型言語で設定を記述するのか」というタイトルで登壇しました。

登壇の経緯や資料作成中に考えたことを振り返ろうと思います。

# 経緯
Nix を使い始めたきっかけが関数型まつり 2025 でした。
関数型プログラミング（以下 FP）って面白い！と思い、Nix の「純粋関数型パッケージマネージャー」という触れ込みに興味を惹かれました。

去年の自分が聴いたら喜びそうなテーマで話したいと思い、「Nix はなぜ純粋関数型言語を使うの？」にしました。


# 想定聴衆の選定
Nix を知らない人にも興味を持って欲しいと思ったので、Nix 未経験者を想定しました。

また、Nix は往々にして関数型の概念・単語を使って解説される事が多いと個人的に感じています。
知らないツールを知らない言葉で解説するのは、初心者に優しくない（Nix 布教の妨げになる）ので、FP 未経験者を想定しました。

----

Nix / FP 未経験者と定義した結果、スライド作りがとても大変でした。
用語でごまかさずに平易な言葉で表現する縛りプレイ状態になりました。


# 感想
## テーマを広くしすぎた
テーマを深掘りしていくと、Nix の設計思想まで行きつきました。
25 分ではなく、50 分枠にすればよかったなと思いました。

尺の都合で、Nix の「純粋関数的な思想」のみにフォーカスしました。
しかし、Nix 言語の性質・意義の一部しか扱えていないです。

例えば、遅延評価、動的型付けといった特徴も深掘りしたかったです。
遅延評価は無駄な計算を省くため、と予測はできます。
しかし、これを深掘りするには、Nix 式の評価・derivation の生成/ビルドのタイミング、等を正確に把握すべきかと思います。
とても長い話になりそうです。


## FP の知識が足りてなかった
Nix の設計思想を考えていくと、副作用があるビルド処理をどうやって純粋関数的なモデルに近づけるか？、がテーマになるなと思いました。

筆者は F# を最近使い始めたばかりです。
モナドも名前は聞いた事がある、といったレベルです。

FP で副作用がある処理で悪戦苦闘した経験があれば、より考察が捗ったかもしれません。
FP をするモチベが上がった気がします。

FP ガチ勢から見た Nix の話を聞いてみたいな、と思いました。


# 余談
Zenn で Nix の記事や本を書いているのは、Nix を普及させる一助になりたいからです。

発表の質疑にて、関数型まつり 2026 の Web サイト開発に Nix を導入したスタッフさんからコメントを頂いたのは感慨深いものでした。
Nix の知名度が上がることで、プロジェクトで Nix を使おうと提案しやすい土壌ができていくと良いなと思いました。
