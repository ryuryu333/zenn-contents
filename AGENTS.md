# Zenn 記事作成ルール（ryu style）

このリポジトリでは、過去記事の文体・構成を踏襲して記事を書く。

## 文体・トーン
- 日本語で、です/ます調。読み手に寄り添う説明。自分語りは最小限にしつつ「私は〜しました」「本記事では〜します」を使う。
- 断定しすぎない。推測/未検証は明示（例: 「推測です」「未検証です」「間違っている可能性があります」）。
- 強調は **太字** を多用し、要点・注意・結論を太字で押さえる。
- 読者を想定した前置きや注意喚起を入れる（例: 「興味がある方だけ」「すでに知っている方は流し読み」）。
- 感情に訴えるストーリー調は避け、技術的な事実と検証に軸足を置く。
- 想定読者はエンジニア。客観性と正確性を優先し、根拠と検証結果を明示する。

## 典型構成
- 先頭に YAML フロントマター（title/emoji/type/topics/published/published_at）。
- 見出し構成は以下を基本形にする。
  - # はじめに
  - # 想定読者（必要なら）
  - # 検証環境（必要なら。長い場合は :::details）
  - # 参考資料（必要なら）
  - # 本題（課題 → 試したこと → 解決 → 深掘り の順が多い）
  - # おわりに
- 手順は番号付きの大見出し（例: # 1.〜）で進行し、サブセクションは 1.1, 1.2 形式。

## 見せ方・装飾
- コードブロックは ```lang:label 形式（例: ```bash:bash, ```nix:flake.nix）。
- 重要な補足や注意は Zenn の :::message / :::message alert を使う。
- 長いログや補足は :::details で折り畳む（タイトルを付ける）。
- 画像は本文中に挿入し、キャプション相当の短い説明を添える。

## 文章の癖（寄せ方）
- 段落は短めでテンポ良く。
- 「つまり」「言い換えると」「ここで」などの接続語を使って論理をつなぐ。
- 仕様や公式情報は引用ブロック（>）＋参照 URL で示す。
- 参照 URL は素の URL で書くことが多い（Markdown リンクも使うが少なめ）。
- 読者の行動に合わせてコマンド提示 → 実行結果 → 解説、の並びを意識する。

## 内容面のルール
- 実際に確認したコマンド結果・バージョンを載せる（検証環境や実行ログ）。
- 既知の問題やデメリットにも触れる（「この場合は〜なので注意」）。
- 内部挙動の解説は「ソースコードを読む → 該当箇所抜粋 → 解釈」の流れで書く。
- 自分の推測や未確証の内容は強調付きで注意書きを入れる。

## 記事内の型（よく使うフレーズ）
- 「本記事では〜を紹介します」
- 「〜という問題に直面しました」
- 「〜を確認してみます」
- 「以下は例です」
- 「詳細は長いので折り畳み」
- 「興味がある方は〜をご覧ください」

## NG
- カジュアルすぎる砕けた口調、絵文字連発。
- 根拠のない断言。
- まとめなしで終わる。
